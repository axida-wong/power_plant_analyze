### 一、环境依赖安装与部署方案

系统后端基于 FastAPI 构建，需要安装高性能科学计算及工业数据解析依赖。

- **第一步：安装 Python 基础环境**

请确保系统已安装 Python 3.10 或更高版本

- **第二步：一键安装核心依赖包**

在主控服务器项目根目录下打开终端或命令提示符（CMD），运行以下命令：

`C:\Users\axida wong> pip install fastapi uvicorn nptdms numpy scipy requests httpx`

![](./pictures/屏幕截图%202026-05-26%20010259.png)

- **第三步：边缘端大模型环境配置**

确保你的LLM大模型，已经运行并开启了web服务。可用以下命令查看LLM模型运行状态。

`axida@runix:~ $ sudo systemctl status llama.service`

![](./pictures/屏幕截图%202026-05-26%20010751.png)

或者用你的浏览器访问运行着LLM模型的主机的IP和端口，查看web服务是否正常。

![](./pictures/屏幕截图%202026-05-26%20010823.png)

### 二、运行程序

---

1. 确保LLM主机的 llama-server 已成功拉起，并在 8080 端口保持监听。

2. 确认原始数据存在：检查项目根目录下是否存在 database/ 文件夹，且内含有 .tdms 工业测试数据文件。

3. 拉起主控后端：在主控服务器根目录下运行：
`D:\my-data\newcode\power_plant> python main.py`

3. 打开前端浏览器：使用 Chrome 或 Edge 浏览器访问 http://127.0.0.1:8000 即可切入可视化监控大屏。

### 三、系统核心功能页面全景功能解析

---

系统开启网页后，包含三大互联互通的功能板块：

1. **火电厂设备实时监测系统主界面 (index.html)**

作为系统的“中央总控大屏”，承载高并发的实时流呈现：
![](./pictures/屏幕截图%202026-05-26%20001001.png)
- 多监测点测点无缝路由（左侧边栏）：系统动态读取 main.py 里的多通道映射字典（NAME_MAP），自动渲染诸如 1#发电机-驱动端振动、2#引风机-机壳振动 等在线物理测点。支持点选切换，波形秒级无缝无感重绘。

![](./pictures/屏幕截图%202026-05-26%20012722.png)

- 时域原始波形动态图（右上卡片）：基于 ECharts 的高性能线条绘制，实时展现高频采集上来的原始物理震荡波形。通过观察波形的“削峰、毛刺、冲击”，工程师能直观感知设备是否存在剧烈的机械摩擦。
![](./pictures/屏幕截图%202026-05-26%20012804.png)
- 频域能量分布分布 FFT 图（右中卡片）：高频原始波形经 scipy 实时就地进行快速傅里叶变换，将复杂的杂乱时域信号拆解为明晰的“频率-能量”频谱柱状图。直接暴露出工频、倍频等具体的动力学故障特征。
![](./pictures/屏幕截图%202026-05-26%20012809.png)
- 实时状态与累计异常看板（右下卡片）：高精度展示当前测点的精确 RMS 有效值。并且右下角内置全厂累计报警频次看板，一旦某个通道的振动值超过了系统设定的安全红线，报警数字实时闪烁翻滚。
![](./pictures/屏幕截图%202026-05-26%20012825.png)
2. **历史异常辅助分析中心 (analysis.html)**

用于在系统发生突发故障后进行设备状态演变追溯及管理汇报：
![](./pictures/屏幕截图%202026-05-26%20001006.png)
- 专属通道动态过滤器：允许工程师按具体的设备测点通道（如 9232/ai0）进行精准过滤，彻底告别大杂烩式的混乱日志。
- 双图表对照体系：
    1. 历史 RMS 恶化演变趋势折线图：将该设备过去所有触发报警的数据串联成时间走势图，图中高亮标记出预警线（3.5）与危险线（6.5），设备是一步步“慢性劣化”还是一瞬间“突发崩溃”一目了然。
    2. 全厂报警频次对比柱状图：全局比对全厂所有设备的报警占比，一眼揪出谁是全厂当前最不稳定的“故障大户”。
![](./pictures/屏幕截图%202026-05-26%20013045.png)
3. **大模型高级专家核心诊断报告 (ai_report.html)**

系统最核心的智能化出口，是边缘 AI 技术的落地体现：
![](./pictures/屏幕截图%202026-05-26%20001500.png)
- 离线AI大语言模型分析：用户点击“🔄 重新生成诊断报告”按钮，前端发出 fetch 请求，主控后端秒级响应并启动“工业级清洗保护架构”。
- 后端向大模型发送异常数据：后端自动从 SQLite 中精准捞取最近且最危险的核心异常轨迹。利用 Python 将复杂的表格、时间戳、英文字母、竖线等大模型不爱读、且容易导致注意力溃散的特殊符号彻底洗净，“脱水”提炼为一行极其顺滑纯正的中文语义叙述（例如：“发电机驱动端发生振动过大，值高达6.8；引风机机壳发生晃动，值达到5.1...”）。
